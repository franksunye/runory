## Decision

**PASS for the v0.6 local Foundation gate.**

The production-shaped local dataset, bounded projections, Command paths, and
20/50-way write contention meet the internal budgets. This is architecture
evidence, not a public hosted-service SLA: remote libSQL and deployed
application/network latency must be measured when that production topology is
selected.

## Reproduction

```text
BENCHMARK_COMMIT=<commit> pnpm benchmark:v06:foundation
```

The script creates an isolated database at
`/tmp/runory-v06-foundation-benchmark.db` and replaces it on each run.

| Field | Value |
| --- | --- |
| Runtime | Node.js 25.8.0, `tsx` |
| Application region | local |
| Database region | local |
| Database mode | SQLite/libSQL local file |
| Dataset | 50 users; 10,000 work orders; 25,000 schedules; 50,000 activity entries; 10,000 form submissions; 5,000 work items |
| Samples | 30 warm-path samples per latency scenario |

## Results

| Scenario | p50 | p95 | p99 | Budget | Result |
| --- | ---: | ---: | ---: | ---: | --- |
| Simple state Command | 0.85 ms | 1.01 ms | 1.10 ms | p95 ≤ 200 ms | pass |
| Create Visit and assign + immediate projection read | 3.90 ms | 9.43 ms | 9.85 ms | p95 ≤ 500 ms | pass |
| Planning week | 3.11 ms | 3.29 ms | 6.91 ms | p95 ≤ 700 ms | pass |
| My Work first page | 0.51 ms | 0.54 ms | 0.57 ms | p95 ≤ 300 ms | pass |
| Activity first page | 0.48 ms | 0.50 ms | 0.54 ms | p95 ≤ 300 ms | pass |

The complex Command check reads the Visit/Schedule projection immediately
after every commit. All 31 runs, including warm-up, observed the new projection
without a stale read.

The script verifies each final aggregate version and the corresponding
successful Command Execution, Domain Event, and Audit row. The figures are not
inferred from fulfilled Promises alone.

## Query Plans and Bounds

- Planning uses `idx_schedule_entries_time` for workspace and start-time
  lookup.
- Activity uses `platform_idx_audit_logs_workspace_created`.
- Planning reads are capped at 2,000 rows; multi-resource requests are capped
  at 100 resources and 2,000 merged entries.
- Activity reads are capped at 500 rows.
- My Work retains its existing cursor pagination and 100-row maximum.

## Deferred Topology Evidence

Hosted application-to-database region latency is not measurable until a
deployment topology is selected. Owner: Engineering. Target: the first hosted
v0.6 deployment candidate. This does not block the Foundation architecture
gate because the local run validates query shape, dataset growth, write
contention, atomicity, and projection freshness; it does block any future
public latency SLA.
