# Runory Website Redesign Specification

Status: Proposed

## Purpose

Redesign the public Runory website from a project/product explanation page into a focused B2B SaaS marketing website.

The website goal is not to fully educate visitors. Deep product, technical, and strategic content belongs in Resources, Blog, and Documentation.

The website should communicate value quickly and convert qualified visitors into pilot conversations.

## Brand Direction

The website shares the Runory brand DNA with existing presentations and architecture documents, but it should not copy the PPT style.

Principles:

- Same brand identity, different expression layer
- Premium, simple, credible, product-oriented
- Less text density
- More product surfaces and real UI
- Avoid generic AI visual language

## Core Positioning

Runory is an Agent-native field service operating system that combines:

- CRM Pack
- Sales Pack
- FSM Pack
- Omnichannel Intake (Voice, SMS, Web)

into one commercial service operation platform.

## Homepage Structure

1. Hero

Message:

Runory — The field service operating system for the Agent era.

Supporting message:

CRM, Sales, and FSM unified into one adaptive platform for service businesses.

CTA:

- Start a Pilot
- Explore Product

Visual placeholder:

Product workspace + mobile field app + Agent interaction.

2. The Problem

Three concise problems:

- Disconnected tools
- Manual operations
- Rigid systems

3. Why Runory

Three differentiators:

- One operating system for service businesses
- External Super Agent first
- Adaptive runtime without rebuilding software

4. Product Experience

Show:

- Operations Workspace
- Mobile Field Experience
- Agent Interface

5. End-to-End Service Journey

Intake → CRM → Sales → FSM → Payment → Retention

6. Business Outcomes

- Faster deployment
- Less administration
- Better visibility
- Continuous adaptation

7. Focused Pilot

Message:

Start small. Mature use cases can often launch in 1–2 weeks.

1. What Runory is.
