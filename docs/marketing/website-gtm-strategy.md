# Runory Website GTM Strategy

Status: Proposed  
Date: 2026-07-15  

## Purpose

The Runory website is the primary marketing and conversion surface. It should not replicate product proposals, architecture documents, or engineering references. Its purpose is to communicate value quickly, establish differentiation, and convert qualified SME field service customers into pilot conversations.

## Positioning

Runory is a field service operating system for the Agent era.

Runory combines:

- CRM Pack
- Sales Pack
- FSM Pack
- Omnichannel Intake (Voice, Phone, SMS, Web)
- External Super Agent operation

into one adaptive business system for service companies.

## Website vs Presentation Materials

The website and enterprise proposal share the same brand DNA but serve different purposes.

### Website

Goal:

- Identify the problem
- Communicate product value
- Demonstrate product experience
- Generate qualified leads

Style:

- concise
- visual
- product-oriented
- conversion-focused

### Proposal / Architecture Documents

Goal:

- explain strategy
- prove technical credibility
- support enterprise discussions

Style:

- detailed
- analytical
- architectural

## Homepage Principles

The homepage should answer:

1. What is Runory?
2. Why is it different?
3. How does it work?
4. What can customers achieve?
5. How can customers start?

## Recommended Homepage Structure

1. Hero

Field service operating system for the Agent era.

2. Market Shift

Traditional software adds AI internally. Runory is designed for external Super Agents.

3. Product Difference

- Unified CRM + Sales + FSM
- Agent-native operation
- Adaptive runtime

4. Product Experience

Show real product surfaces:

- Operations Workspace
- Mobile Field App
- Agent Interface

5. End-to-End Service Journey

Intake → CRM → Sales → FSM → Payment → Retention

6. Business Outcomes

- Faster implementation
- Less administration
- Better operational control
- Continuous adaptation

7. Focused Pilot

A mature use case can typically launch in 1–2 weeks.

8. CTA

Start a focused pilot.

## Content Strategy

The website should remain concise. Deep explanations move to Resources:

- Product guides
- Architecture articles
- Agent-native software concepts
- FSM industry insights
- Technical documentation

## Design Direction

Maintain Runory brand DNA:

- warm ivory background
- near black typography
- orange accent
- premium minimalism

Add website-specific elements:

- product screenshots
- interactive demonstrations
- customer scenarios
- subtle motion

Avoid:

- AI cliché visuals
- excessive gradients
- dense text blocks
- architecture-heavy homepage sections
